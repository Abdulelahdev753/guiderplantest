# GuiderPlan — منصة الأدلة السياحية الرقمية

> منصة ويب متكاملة لبيع وتوزيع أدلة السفر الرقمية بالتعاون مع وكالة نجوم السفر والسياحة

---

## نبذة عن المشروع

**GuiderPlan** منصة تجارة إلكترونية متخصصة في بيع أدلة السفر الرقمية (PDF) لوجهات سياحية عالمية متعددة. تتيح المنصة للمستخدم شراء دليل سفر بخطوات بسيطة، إتمام الدفع عبر بوابة **StreamPay**، ثم تحميل الملف مباشرةً بعد تأكيد الدفع. كما تضم المنصة قسماً خاصاً لحجز البرامج السياحية المتكاملة بالتعاون مع وكالة نجوم السفر والسياحة.

المنصة مزدوجة اللغة (عربي / إنجليزي) وتدعم الاتجاه من اليمين لليسار بشكل كامل.

---

## الميزات الرئيسية

- **متجر أدلة السفر**: بيع ملفات PDF لوجهات مثل لندن، برشلونة، براغ، أمستردام، شمال إيطاليا، والباقة الاقتصادية (بودابست + براغ).
- **نظام دفع آمن**: تكامل مع بوابة **StreamPay** السعودية لإنشاء روابط دفع آمنة.
- **تحميل مؤمَّن بالرمز**: بعد إتمام الدفع يُرسل رابط تحميل فريد صالح لـ 72 ساعة.
- **لوحة الحجز السياحي**: نموذج تواصل مع الوكالة السياحية يُرسل الطلب عبر البريد الإلكتروني ويحفظه في قاعدة البيانات.
- **تعدد اللغات**: واجهة كاملة بالعربية والإنجليزية مع دعم RTL/LTR.
- **إشعارات البريد الإلكتروني**: إرسال إشعارات فورية لفريق العمل عند كل طلب حجز جديد عبر **Resend**.
- **تصميم متجاوب**: يعمل بسلاسة على الجوال والحاسوب.

## التقنيات المستخدمة

### الواجهة الأمامية (Frontend)
| التقنية | الوصف |
|---|---|
| Next.js 19 | إطار React للتطبيقات الحديثة |
| TypeScript | الكتابة الصارمة للكود |
| Tailwind CSS 4 | تصميم سريع ومتجاوب |

| GSAP | مكتبة الحركات والانيميشن |
| Lucide React | أيقونات SVG |

### الواجهة الخلفية (Backend)
| التقنية | الوصف |
|---|---|
| Express.js 5 | خادم API سريع وخفيف |
| TypeScript | الكتابة الصارمة للكود |
| Supabase | قاعدة بيانات PostgreSQL + تخزين الملفات |
| StreamPay | بوابة الدفع السعودية |
| Resend | خدمة إرسال البريد الإلكتروني |
| Docker | حاوية النشر |

## نقاط API الرئيسية

| الطريقة | المسار | الوصف |
|---|---|---|
| `POST` | `/api/payment/create-link` | إنشاء رابط دفع عبر StreamPay |
| `POST` | `/api/webhooks/streampay` | استقبال تأكيد الدفع من StreamPay |
| `GET` | `/api/download/status` | التحقق من حالة الدفع قبل التحميل |
| `GET` | `/api/download` | تحميل ملف PDF بعد إتمام الدفع |
| `POST` | `/api/agency-contact` | إرسال طلب تواصل مع الوكالة السياحية |
| `POST` | `/api/booking/request` | إرسال طلب حجز برنامج سياحي |

## آلية عمل نظام الشراء والتحميل

```
1. المستخدم يختار الدليل ← 2. إدخال الاسم والجوال
      ↓
3. الباكيند ينشئ سجلاً في Supabase برمز فريد (UUID)
      ↓
4. إنشاء رابط دفع عبر StreamPay API
      ↓
5. المستخدم يكمل الدفع في StreamPay
      ↓
6. StreamPay يُرسل Webhook → الباكيند يُحدّث حالة الطلب إلى "completed"
      ↓
7. المستخدم يُعاد توجيهه لصفحة التحميل برمز الجلسة
      ↓
8. تحميل PDF مباشرة من Supabase Storage (صالح 72 ساعة)
```

## النشر

المشروع يعمل كتطبيق موحد: الباكيند يخدم ملفات Next.js المُصدَّرة بشكل ثابت ويتولى جميع نقاط API.

```bash
# بناء الواجهة الأمامية
cd frontend && npm run build

# تشغيل الخادم (يخدم الفرونت والـ API معاً)
cd backend && npm start
```

أو عبر Docker:
```bash
docker build -t guiderplan .
docker run -p 5001:5001 guiderplan
```
